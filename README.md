# DosCrack


## Суть задания

Целью задания является взлом несложной dos'овской программы. Я получаю .com файл
с программой, которая запрашивает пароль, проверяет его и, в случае совпадения,
выводит строку 'access granted', в противном же - 'access denied'. Нужно добиться
вывода 'access granted', не зная пароль.

Программа содержит две специально заложенные уязвимости, которые позволяют мне
изменить её поведение и добиться 'access granted' без ввода правильной комбинации.
Использование Turbo Debugger делает возможным детально изучить код и выявить
вышеупомянутые уязвимости.

## Поиск второй уязвимости

Я замечаю, что программа не контролирует вводимое мною количество символов. Это позволяет мне воспользоваться переполнением
буфера для взлома. Хеш-функция берет некоторую информацию из сегмента памяти, расположенного между паролем и выводимыми для
пользователя сообщениями...

<img width="1009" height="306" alt="image" src="https://github.com/user-attachments/assets/123a33dc-9a2f-4cc7-8c68-d72c3e3d616c" />

...и кладёт их в AX при каждой итерации цикла.

<img width="992" height="271" alt="image" src="https://github.com/user-attachments/assets/06499051-ca2b-495e-82cb-375356c6777b" />

Это единственные изменения, происходящие с регистром AX. Поэтому следующее, что я попробовала - замена всех данных между паролем
и сообщениями случайными, но одинаковыми символами. Они будут положены в AX, и такие же попадут в BX(поскольку в той же области
лежит хеш). 

<img width="936" height="98" alt="image" src="https://github.com/user-attachments/assets/a7a5fbfc-a4a8-401a-a5ba-26f0a4ad5790" />

## Заключение

Безусловно, существует несколько способов взломать программу, но идея о переполнении буфера - одна из самых очевидных,
поэтому в процессе взлома я искала что-то, что позволило бы при помощи переполнения затереть важные данные или даже
расположить свой код внутри программы.
